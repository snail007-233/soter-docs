Soter代码规范
  1. 变量名，方法名，函数名，参数名，采取驼峰命名方式。比如userId
  2. 命名采用英文完整单词不能缩写，取最接近功能的单词。比如：userScoreLevel
  3. 常量名，静态变量名，采取全部大写加下划线分割多个单词的命名方式。比如：MAX_ID
  4. classes类目录下的：文件名、类名、文件的位置，三者是对应的，大小写敏感。
    1.比如有文件：classes/Controller/User.php
    那么User.php文件里面的类名就应该是：Controller_User
    2.比如有文件：classes/Controller/Vip/User.php
    那么User.php文件里面的类名就应该是：Controller_Vip_User,也就是下划线代表着文件夹的分隔符。
    3.比如有文件：classes/Model/User.php
    那么User.php文件里面的类名就应该是：Model_User
    4.比如有文件：classes/MyLib/User.php
    那么User.php文件里面的类名就应该是：MyLib_User
  5. 类库目录libray/classes下的文件名和类名是对应的。
    比如有文件：libray/classes/User.php
    那么User.php文件里面的类名就应该是：User
  6. 代码采用tab缩进，一个tab占用8个英文半角空格宽度
